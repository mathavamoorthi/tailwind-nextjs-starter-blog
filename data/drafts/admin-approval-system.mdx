#  Admin Approval System

This guide explains the **admin approval system** implemented in your Tailwind Next.js blog, which allows multiple users to create content while maintaining quality control through admin approval.

##  **System Overview**

### **📋 Workflow:**
1. **Users create drafts** → Posts saved to `/data/drafts/` with `status: draft`
2. **Admin reviews** → Access admin dashboard at `/admin`
3. **Admin approves/rejects** → Posts moved to `/data/blog/` or marked as rejected
4. **Published content** → Only approved posts appear on the public blog

### ** Authentication:**
- **Editor Users**: Can create and edit drafts (existing `EDITOR_USERS`)
- **Admin Users**: Can approve/reject drafts (new `ADMIN_USERS`)

##  **Environment Variables Setup**

Add these to your `.env.local` and Vercel environment variables:

```bash
# Existing Editor Users (can create drafts)
EDITOR_USERS=username1:password1:author1,username2:password2:author2

# New Admin Users (can approve/reject drafts)
ADMIN_USERS=admin1:password1,admin2:password2

# Other existing variables
BLOB_READ_WRITE_TOKEN=your_vercel_blob_token
GITHUB_TOKEN=your_github_personal_access_token
GITHUB_OWNER=your_github_username
GITHUB_REPO=your_repository_name
```

##  **How It Works**

### **1. Draft Creation (Users)**
```
User creates post → Saves as draft → Stored in /data/drafts/ → Status: draft
```

**Features:**
- ✅ **Multiple users** can create drafts simultaneously
- ✅ **Image processing** works the same (Vercel Blob → GitHub)
- ✅ **Draft status** automatically added to frontmatter
- ✅ **Timestamps** track creation and updates

### **2. Admin Review (Admins)**
```
Admin logs in → Views dashboard → Reviews drafts → Approves/rejects
```

**Admin Dashboard Features:**
-  **Draft overview** with status, author, creation date
-  **Content preview** with excerpts
-  **One-click approval** moves to published blog
-  **Rejection with feedback** keeps in drafts
-  **Conflict detection** for duplicate slugs

### **3. Publishing Process (Admins)**
```
Admin approves → Draft moved to /data/blog/ → GitHub commit → Vercel auto-deploy
```

**Approval Process:**
-  **Automatic GitHub commit** when approved
-  **Vercel auto-deployment** triggered
-  **Page revalidation** for immediate updates
-  **Draft cleanup** (deleted after approval)

##  **Access Control**

### **Editor Access (`/editor`)**
- **Authentication**: `EDITOR_USERS` environment variable
- **Permissions**: Create, edit, save drafts
- **Access**: Basic auth with username:password

### **Admin Access (`/admin`)**
- **Authentication**: `ADMIN_USERS` environment variable  
- **Permissions**: View, approve, reject drafts
- **Access**: Basic auth with username:password

##  **Admin Dashboard Features**

### **Draft Management**
-  **Draft listing** with status indicators
-  **Author information** for each draft
-  **Creation/update timestamps**
-  **Content excerpts** for quick review
-  **Conflict warnings** for duplicate slugs

### **Approval Actions**
-  **Approve**: Move to published blog, commit to GitHub
-  **Reject**: Mark as rejected, add feedback
-  **Feedback system**: Optional rejection comments
-  **Real-time updates**: Dashboard refreshes after actions

### **Status Tracking**
-  **Draft**: Awaiting review
-  **Rejected**: Rejected with feedback
-  **Approved**: Published and live

##  **API Endpoints**

### **`GET /api/admin/drafts`**
Lists all draft posts for admin review.

**Headers:**
```
Authorization: Basic <base64-encoded-admin-credentials>
```

**Response:**
```json
{
  "drafts": [
    {
      "slug": "my-draft-post",
      "title": "My Draft Post",
      "authors": ["author1"],
      "status": "draft",
      "createdAt": "2024-01-15T10:30:00Z",
      "updatedAt": "2024-01-15T10:30:00Z",
      "isConflict": false,
      "excerpt": "This is the beginning of the post..."
    }
  ],
  "total": 1
}
```

### **`POST /api/admin/approve`**
Approves or rejects a draft post.

**Headers:**
```
Authorization: Basic <base64-encoded-admin-credentials>
Content-Type: application/json
```

**Body:**
```json
{
  "slug": "my-draft-post",
  "action": "approve|reject",
  "feedback": "Optional feedback for rejection"
}
```

**Response:**
```json
{
  "success": true,
  "message": "Post approved and published successfully"
}
```

##  **Workflow Examples**

### **Example 1: Successful Approval**
1. **User creates draft**: `my-awesome-post.mdx`
2. **Admin reviews**: Sees draft in dashboard
3. **Admin approves**: Clicks "Approve" button
4. **System processes**: 
   - Moves file from `/data/drafts/` to `/data/blog/`
   - Commits to GitHub
   - Triggers Vercel deployment
   - Updates live blog

### **Example 2: Rejection with Feedback**
1. **User creates draft**: `needs-improvement.mdx`
2. **Admin reviews**: Sees issues with content
3. **Admin rejects**: Clicks "Reject" with feedback
4. **System processes**:
   - Updates draft status to `rejected`
   - Adds feedback to frontmatter
   - Keeps in drafts for user to revise

## 🔧 **Troubleshooting**

### **Common Issues**

**Admin can't access dashboard:**
- Check `ADMIN_USERS` environment variable
- Verify credentials format: `username:password`
- Ensure middleware is protecting `/admin` routes

**Drafts not appearing:**
- Check `/data/drafts/` directory exists
- Verify draft files have correct frontmatter
- Check GitHub permissions for draft commits

**Approval not working:**
- Verify GitHub token has `repo` scope
- Check admin credentials are correct
- Ensure file permissions allow moving files

### **Debug Commands**
```bash
# Check if drafts directory exists
ls -la data/drafts/

# Verify admin environment variable
echo $ADMIN_USERS

# Test admin API endpoint
curl -u admin:password http://localhost:3000/api/admin/drafts
```
