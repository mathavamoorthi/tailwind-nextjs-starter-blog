## Overview

This project is a **production-ready blog** built on the Tailwind Next.js Starter Blog template, enhanced with a sophisticated **staging/production image workflow**. It provides:

- **Instant image preview** during editing (staging)
- **Optimal production performance** with static images
- **Automatic deployment** via Vercel
- **Professional editing experience** with real-time MDX preview

## 🔧 Setup Guide

### **1. Prerequisites**

- Node.js 18+ 
- Yarn package manager
- GitHub account
- Vercel account

### **2. Clone and Install**

```bash
# Clone the repository
git clone https://github.com/your-username/tailwind-nextjs-starter-blog.git
cd tailwind-nextjs-starter-blog

# Install dependencies
yarn install

# Start development server
yarn dev
```

### **3. Environment Configuration**

Create a `.env.local` file with the required environment variables:

```bash
# Vercel Blob Storage (Required for staging)
BLOB_READ_WRITE_TOKEN=your_vercel_blob_token

# GitHub Integration (Required for production)
GITHUB_TOKEN=your_github_personal_access_token
GITHUB_OWNER=your_github_username
GITHUB_REPO=your_repository_name

# Editor Authentication (Required)
EDITOR_USERS=username:password:author_name
```

##  Environment Variables

### **Required Variables**

| Variable | Description | How to Get |
|----------|-------------|------------|
| `BLOB_READ_WRITE_TOKEN` | Vercel Blob storage token | Vercel Dashboard → Storage → Blob |
| `GITHUB_TOKEN` | GitHub personal access token | GitHub Settings → Developer settings → Personal access tokens |
| `GITHUB_OWNER` | Your GitHub username | Your GitHub profile |
| `GITHUB_REPO` | Repository name | Your repository name |
| `EDITOR_USERS` | Editor authentication | Format: `username:password:author_name` |

### **Optional Variables**

| Variable | Description | Default |
|----------|-------------|---------|
| `GITHUB_BRANCH` | GitHub branch for commits | `main` |
| `NEXT_PUBLIC_SITE_URL` | Your site URL | Auto-detected |

##  API Routes

### **`/api/editor/upload`**
Handles image uploads to Vercel Blob Storage.

**Method:** `POST`  
**Body:** `FormData` with `file` and `slug`  
**Response:** CDN URL for immediate preview

```typescript
// Example response
{
  ok: true,
  url: "https://blob.vercel-storage.com/...",
  filename: "1234567890-image.png",
  message: "✅ Image uploaded to Vercel Blob!"
}
```

### **`/api/editor/save`**
Saves MDX content and processes Blob images.

**Method:** `POST`  
**Body:** `{ frontmatter, body, slug }`  
**Response:** Success status with processing results

```typescript
// Example response
{
  ok: true,
  message: "Blog post committed to GitHub successfully! 2 images processed and committed. Vercel will auto-deploy.",
  images: {
    processed: ["image1.png", "image2.jpg"],
    failed: [],
    total: 2
  }
}
```

### **`/api/editor/process-blob-images`**
Processes Vercel Blob URLs and commits images to GitHub.

**Method:** `POST`  
**Body:** `{ mdxContent, slug }`  
**Response:** Processed content with local image paths

## Editor Usage

### **1. Access the Editor**

Navigate to `/editor` and authenticate with your credentials.

### **2. Create a New Post**

1. **Fill in frontmatter:**
   - **Title**: Your post title
   - **Date**: Publication date
   - **Tags**: Comma-separated tags
   - **Summary**: Brief description
   - **Authors**: Your author name

2. **Write your content:**
   - Use standard Markdown syntax
   - Add images by dragging and dropping
   - Preview updates in real-time

### **3. Add Images**

**Method 1: Drag & Drop**
- Drag image files directly into the editor
- Images upload to Vercel Blob automatically
- CDN URLs appear instantly in preview

**Method 2: Paste from Clipboard**
- Copy an image to clipboard
- Paste directly into the editor
- Automatic upload and preview

### **4. Publish Your Post**

1. Click **"Save MDX"** button
2. System automatically:
   - Downloads images from Vercel Blob
   - Commits images to GitHub
   - Updates MDX URLs to local paths
   - Commits MDX content to GitHub
3. Vercel automatically deploys the changes

##  Deployment

### **Vercel Deployment**

1. **Connect Repository:**
   - Go to Vercel Dashboard
   - Import your GitHub repository
   - Configure build settings

2. **Set Environment Variables:**
   - Add all required environment variables
   - Ensure `BLOB_READ_WRITE_TOKEN` is set
   - Configure GitHub integration

3. **Enable Auto-Deployment:**
   - Vercel automatically deploys on GitHub commits
   - No additional webhook configuration needed

### **Build Configuration**

```json
{
  "buildCommand": "yarn build",
  "outputDirectory": "out",
  "installCommand": "yarn install"
}
```

### **Custom Domain (Optional)**

1. Add custom domain in Vercel Dashboard
2. Configure DNS records
3. Enable HTTPS automatically
